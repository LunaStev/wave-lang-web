# تشخیص خطا

کامپایلر Wave خطاها را همراه با کد (`E####`) و همچنین موقعیت/متن/پیشنهاد حل‌آنها به یکباره نشان می‌دهد.

## فرمت خروجی

قالب پایه به صورت زیر است.

```text
error[E3001]: اعتبارسنجی معنایی شکست خورد: استفاده از شناسه `x` اعلام نشده
  --> file.wave:2:18
 1 | fun main() {
 2 |     println("{}", x);
   |                  ^ در این حوزه یافت نشد
   = context: اعتبارسنجی معنایی
   = help: مشکلات تغییرپذیری، محدوده و اعتبار بیانی را برطرف کنید
```

موارد خروجی:

- `error[E....]`: کد خطا و خلاصه
- `--> file:line:column`: موقعیت خطا
- بلوک کد منبع + هایلایت caret(`^`)
- `زمینه`، `مورد انتظار`، `یافت شده`، `توضیح`، `راهنما`، `پیشنهاد`

## کد خطای اصلی

- `E1001` نویسه غیرمنتظره
- `E1002` توضیح بلوک بسته نشده
- `E1003` رشته بسته نشده
- `E1004` فرار رشته اشتباه
- `E1005` لیتِرال نویسه اشتباه
- `E1006` قالب لیتِرال عددی اشتباه
- `E2001` خطای تجزیه‌کننده در نحو
- `E3001` خطای تحلیل معنایی (اعتبارسنجی معنایی)
- `E3102` تخصیص `null` به غیر اشاره‌گر
- `E3201` کاهش ضمنی عدد صحیح ممنوع است
- `E9001` خطای داخلی در تولید کد Backend

## نشان‌گذاری موقعیت منبع حتی در خطای Backend

حتی اگر در مرحله تولید کد (LLVM) یک وحشت داخلی رخ دهد، تا حد امکان موقعیت واقعی فراخوانی/اعلام را استنتاج و نمایش می‌دهد.

```text
خطا[E9001]: خطای داخلی کامپایلر در طول تولید کد (llvm-ir-generation)
  --> test.wave:12:9
   = یافت شده: تابع 'foo' پیدا نشد
   = توضیح: موقعیت منبع از نام تابع نامشخص در وحشت Backend استنتاج شده است
```

در مواردی که استنتاج موقعیت ممکن نیست، موقعیت پشتیبان استفاده می‌شود و این واقعیت در `یادداشت` نیز نمایش داده می‌شود.
