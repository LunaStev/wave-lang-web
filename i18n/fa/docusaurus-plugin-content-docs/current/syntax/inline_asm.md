# اسمبلر داخلی

## معرفی

اسمبلر درون خطی Wave با استفاده از `asm { ... }` نوشته می‌شود.
در کد Wave می‌توان مستقیماً ثبات‌ها، حافظه و مسیرهای فراخوانی سیستم را کنترل کرد.

هدف‌های پشتیبانی شده فعلی:

- لینوکس `x86_64`
- لینوکس `aarch64`
- مک‌اواس (Darwin) `arm64`
- استندالون `x86_64`
- استندالون `aarch64`
- استندالون `riscv64`

Windows و هدف‌های 32 بیتی هنوز پشتیبانی نمی‌شوند.

---

## فرم پایه‌ای

`asm` هم به صورت **عبارت (statement)** و هم به صورت **تعبیر (expression)** قابل استفاده است.

```wave
asm {
    "instruction"
    in("constraint_or_reg") value
    out("constraint_or_reg") target
    clobber("item")
}
```

مؤلفه‌ها:

- خط رشته: دستور زبان واقعی اسمبلر
- `in(...)`: عملوند ورودی
- `out(...)`: عملوند خروجی
- `clobber(...)`: اشاره به ثبات/وضعیت/حافظه‌ای که نابود می‌شود

---

## عبارت `asm` (Statement)

در صورت نیاز به عدم برگشت مقدار، به عنوان جمله عادی استفاده می‌شود.

```wave
var ret: i64 = 0;
asm {
    "mov rax, 1"
    "syscall"
    in("rdi") 1
    in("rsi") msg_ptr
    in("rdx") 20
    out("rax") ret
}
```

`out(...)` می‌تواند دارای چندین تعداد باشد.

---

## تعبیر `asm` (Expression)

می‌توان از آن به عنوان عبارتی برای تولید مستقیم مقدار استفاده کرد.

```wave
var result: i64 = asm {
    "mov rax, 123"
    out("rax") result
};
```

توجه:

- تعبیر `asm` فقط **یک `out(...)`** را مجاز می‌داند.

---

## قید `in(...)` / `out(...)`

رشته های `in("...")`، `out("...")` یکی از دو مورد زیر است.

1. ثبات خاص

- مثال: `"rax"`, `"rdi"`, `"x0"`, `"w1"`, `"a0"`, `"t0"`, `"x10"`

2. کلاس قید (constraint class)

- مثال: `"r"`, `"m"`, `"rm"`

مثال:

```wave
in("r") &buf
out("rax") ret
```

هدف خروجی (`out(...) target`) به عنوان الگوی پیشنهادی براساس پیاده‌سازی فعلی توصیه می‌شود.

- متغیر: `out("rax") ret`
- دی‌رفرنس اشاره‌گر: `out("rax") deref p`

---

## `clobber(...)`

`clobber(...)` می‌تواند چندین مورد را به طور همزمان بپذیرد و چندین بار مورد استفاده قرار گیرد.

```wave
asm {
    "xor rax, rax"
    clobber("rax")
    clobber("rcx", "rdx")
    clobber("memory")
}
```

موارد اصلی:

- ثبات: `"rax"`, `"x0"` و غیره
- ویژه: `"memory"`, `"cc"` (تطبیق داخلی به‌خصوص بر اساس هدف)

کامپایلر در حالت محافظه‌کارانهٔ ایمنی، به‌طور خودکار clobber پیش‌فرض را اضافه می‌کند.
(`حافظه`، پرچم/درجه‌بندی cc و غیره؛ در RISC-V freestanding بیشتر `حافظه`)

---

## جانشین عملوند (`$0`, `$1`, ...)

هنگام ارجاع به اپرانده‌ها درون رشته دستور از `$N` استفاده می‌شود.

```wave
اسم {
    "mov QWORD PTR [$0], 777"
    در("r") &buf
    بخشی از حافظه("memor clobber")
}
```

توجه:

- حتی اگر از سبک `%0` استفاده کنید، به‌طور داخلی به سبک `$0` تبدیل می‌شود.

---

## محدوده فعلی پشتیبانی از اپرانده‌های ورودی

مقادیر `in(...)` در حال حاضر از اشکال زیر پشتیبانی می‌کنند.

- شناسه متغیر
- ثابت عدد صحیح
- ثابت رشته‌ای
- `&شناسه`
- `دسترسی غیر مستقیم شناسه`
- ثابت‌های عدد صحیح/شناور منفی

عبارت‌های عمومی پیچیده ممکن است محدود شوند، بنابراین توصیه می‌شود در صورت نیاز آن‌ها را در یک متغیر موقت ذخیره و ارسال کنید.

---

## ملاحظات

اسمبلر درون‌خطی به‌صورت جزئی از حفاظت سیستم نوع گذر می‌کند.
تعیین نادرست رجیستر، تضادهای قید و حذف clobber می‌تواند باعث ایجاد کد اشتباه یا عملکرد نادرست در زمان اجرا شود.

توصیه‌ها:

- اول از همه، ABI هدف و قرارداد فراخوانی را تعیین کنید
- رجیسترهای ورودی/خروجی و clobber را به‌طور واضح مدیریت کنید
- در صورتی که به‌طور مستقیم حافظه را تغییر می‌دهید، `clobber("memory")` را نیز اعلام کنید
