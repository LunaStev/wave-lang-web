# পয়েন্টার

## ওয়েভ স্পষ্ট মেমরি টাইপ মডেল

ওয়েভের পয়েন্টার ডিজাইন **ওয়েভ স্পষ্ট মেমরি টাইপ মডেল** এর উপর ভিত্তি করে।
এই মডেলটি পয়েন্টার ও অ্যারেগুলিকে সিনট্যাক্সিয়াল ট্রিক বা লাইব্রেরি অ্যাবস্ট্রাকশন হিসেবে নয়, **ভাষাগত স্তরে স্পষ্ট মেমরি টাইপ** হিসেবে সংজ্ঞায়িত করার লক্ষ্য।

এই ধরনের ডিজাইনের মাধ্যমে ওয়েভে, পয়েন্টারকে `ptr<T>` আকারে নির্দিষ্ট টাইপ হিসেবে প্রকাশ করা হয়,
এবং এটি একটি নির্দিষ্ট টাইপ `T` এর মান সংরক্ষিত মেমরি ঠিকানা নির্দেশ করে যে টাইপ।
এই অ্যাপ্রোচটি পয়েন্টারকে অপারেটর বা ঘোষণার সিনট্যাক্স হিসেবে নয় বরং টাইপ সিস্টেমের অংশ হিসেবে গ্রহণ করা হয়,
এবং মেমরি গঠনকে আরো স্পষ্ট ও সুসংগতভাবে প্রকাশ করতে সহায়ক।

---

웨이브에서 পয়েন্টার হলো `ptr<T>` আকারে স্পষ্ট টাইপ।
ঠিকানা আহরণ করা হয় `&`, এবং ডেরেফারেন্স করা হয় `deref` ব্যবহার করে।

## ঘোষণা এবং প্রাথমিককরণ

```wave
var x: i32 = 10;
var p: ptr<i32> = &x;
```

পয়েন্টার টাইপগুলি নেস্ট করা যায়।

```wave
var p1: ptr<i32> = &x;
var p2: ptr<ptr<i32>> = &p1;
```

## ডেরেফারেন্স

```wave
var x: i32 = 10;
var p: ptr<i32> = &x;

println("{}", deref p); // 10
deref p = 20;
println("{}", x);       // 20
```

## `null` লিটারেল নিয়মাবলী

`null` হলো একটি **আনুষ্ঠানিক লিটারেল**। এটি একটি শনাক্তকারী নয় এবং এটি একটি ভেরিয়েবলের নাম হিসেবে ব্যবহার করা যাবে না।

মূল নিয়মাবলী:

- `null` কেবল `ptr<T>` লক্ষ্যগুলির জন্যই বরাদ্দ করা যেতে পারে।
- `i32`, `bool`, `array<...>` মত নন-পয়েন্টার টাইপে এটি বরাদ্দ করা যাবে না।
- ইনটিজার লিটারেলস (`0`, `123`, `-1` ইত্যাদি) দিয়ে পয়েন্টার ইনিশিয়ালাইজ করা যাবে না। স্পষ্টভাবে `null` ব্যবহার করুন।

```wave
var p: ptr<i32> = null;
var arrp: ptr<array<i32, 3>> = null;

// var n: i32 = null;  // ত্রুটি
// var b: bool = null; // ত্রুটি
```

## পয়েন্টার আরিথমেটিক

ওয়েভ নিম্নলিখিত পয়েন্টার আরিথমেটিক সমর্থন করে।

- `ptr + int`: GEP ভিত্তিক পয়েন্টার ফরোয়ার্ডিং
- `int + ptr`: একই কার্যক্রম
- `ptr - int`: GEP ভিত্তিক পয়েন্টার ব্যাকওয়ার্ডিং
- `ptr - ptr`: `i64` বাইট ডিফারেন্স ক্যালকুলেশন

বিন্দু:

- `ptr<T> +/- n` চারপাশে `T` এর আকার (`sizeof(T)`) এর ভিত্তিতে স্থানান্তরিত হয়।
- অর্থাৎ, `ptr<i32> + 3` বাইট ভিত্তিতে `+12` স্থানান্তর।

```wave
var base: ptr<i32> = 0x1000 as ptr<i32>;

var p1: ptr<i32> = base + 3; // 0x1000 + 12
var p2: ptr<i32> = 2 + base; // 0x1000 + 8
var p3: ptr<i32> = base - 1; // 0x1000 - 4

var diff: i64 = p1 - base;   // 12 (byte diff)
```

## পয়েন্টার তুলনা

পয়েন্টারগুলি তুলনায় ব্যবহৃত হতে পারে।

```wave
if (p == null) { ... }
if (p != null) { ... }
if (p1 == p2) { ... }
```

## অ্যারের সম্পর্ক

পয়েন্টার অ্যারে:

```wave
var a: i32 = 10;
var b: i32 = 20;
var arr: array<ptr<i32>, 2> = [&a, &b];
println("{} {}", deref arr[0], deref arr[1]);
```

অ্যারে পয়েন্টার:

```wave
var p: ptr<array<i32, 3>> = &[1, 2, 3];
if (p != null) {
    println("{}", deref p[1]);
}
```

## নিরাপত্তা নোট

ওয়েভ বর্তমানে Rust-এর মতো মালিকানা/আয়ুষ্কাল ভিত্তিক পয়েন্টার নিরাপত্তা মডেল নয়।
অতএব, `null` ডেরেফারেন্স স্বয়ংক্রিয়ভাবে অবরুদ্ধ করা হচ্ছে না। প্রস্তাবিত হয় `deref` এর আগে স্পষ্টভাবে একটি `null` চেক যোগ করা হয়।
