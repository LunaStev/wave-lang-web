# ত্রুটি নির্ণয়

Wave কম্পাইলার ত্রুটির কোড (`E####`) সহ সোর্স অবস্থান/প্রসঙ্গ/সমাধান সূচক পর্যন্ত একবারে প্রদর্শন করে।

## আউটপুট বিন্যাস

মৌলিক বিন্যাস টি নিম্নরূপ।

```text
ত্রুটি [E3001]: অর্থবহ যাচাই ব্যর্থ হয়েছে: ঘোষিত নয় এমন শনাক্তকারী `x` এর ব্যবহার
  --> ফাইল.wave:2:18
 1 | ফান প্রধান() {
 2 |     প্রিন্টলন("{}", x);
   |                  ^ এই ক্ষেত্রের মধ্যে পাওয়া যায়নি
   = প্রসঙ্গ: অর্থবহ যাচাই
   = সাহায্য: পরিবর্তনশীলতা, ক্ষেত্র এবং অভিব্যক্তির বৈধতার বিষয়গুলি সমাধান করুন।
```

আউটপুট আইটেম:

- `error[E....]`: ত্রুটি কোড এবং সারসংক্ষেপ
- `--> ফাইল:লাইন:কলাম`: সমস্যার স্থান
- সোর্স ব্লক + ক্যারেট (`^`) হাইলাইট
- `প্রসঙ্গ`, `প্রত্যাশিত`, `পাওয়া গেছে`, `নট`, `সাহায্য`, `প্রস্তাবনা`

## প্রধান ত্রুটি কোড

- `E1001` অপ্রত্যাশিত চিহ্ন
- `E1002` দুর্বোধ্য ব্লক মন্তব্য
- `E1003` অপূর্ণ স্ট্রিং
- `E1004` ভুল স্ট্রিং পালানোর সংকেত
- `E1005` ভুল অক্ষর লিটারেল
- `E1006` ভুল সংখ্যা লিটারেল বিন্যাস
- `E2001` পার্সার সিনট্যাক্স ত্রুটি
- `E3001` অর্থগত বিশ্লেষণ (সেমান্টিক যাচাইকরণ) ত্রুটি
- `E3102` বিয়োগের পয়েন্টারে `null` বরাদ্দ
- `E3201` অপ্রকাশিত পূর্ণসংখ্যা সংকোচন নিষিদ্ধ
- `E9001` ব্যাকেন্ড কোড সৃষ্টির অভ্যন্তরীণ ত্রুটি

## ব্যাকেন্ড ত্রুটিগুলি উৎস অবস্থানকেও প্রদর্শন করে

কোড জেনারেশন (LLVM) ধাপগুলিতে অন্তর্নিহিত প্যানিক ঘটলেও, সম্ভাব্য ক্ষেত্রে আসল কল/ঘোষণা অবস্থান অনুমান করে প্রদর্শন করে।

```text
ত্রুটি [E9001]: কোড জেনারেশন সময় (llvm-ir-generation) কম্পাইলার অভ্যন্তরীণ ত্রুটি
  --> test.wave:12:9
   = লক্ষ্য: ফাংশন 'foo' পাওয়া যায়নি
   = নোট: ব্যাকেন্ড প্যানিকের মধ্যে অবিন্যাসিত ফাংশন নাম থেকে উৎস অবস্থান অনুমান করা হয়েছে
```

অবস্থানের অনুমান সম্ভব না হলে, fallback অবস্থান ব্যবহৃত হয় এবং সেই বিষয়টি `note` এ একসাথে প্রদর্শিত হয়।
