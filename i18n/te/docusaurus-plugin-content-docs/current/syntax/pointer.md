# సూచిక

## Wave స్పష్టమైన మెమరీ రకం నమూనా

Wave యొక్క పాయింటర్ రూపకల్పన **Wave స్పష్టమైన మెమరీ రకం నమూనా**తో ఆధారంగా ఉంది.
ఈ నమూనా పాయింటర్ మరియు అక్కులను ఒక సరళమైన చిట్కా లేక లైబ్రరీ శ్రేణి కాకుండా, **భాషా స్థాయిలో స్పష్టమైన మెమరీ రకం**గా నిర్వచించాలనుకుంటుంది.

ఈ రూపకల్పన ప్రకారం Wave పాయింటర్ ను `ptr<T>` రూప ఆ కర్ణంలో ఆకృతిలో చూపిస్తుంది,
ఇది ప్రత్యేక రకం `T` యొక్క విలువలను కలిగి ఉండే మెమరీ చిరునామాను సూచించే రకంగా స్పష్టం ఉంటుంది.
ఈ పద్ధతిని ప్రదర్శించి నివ్వాకే, తేదీ దృష్టి చిహ్నం కాని, ఇవళ్ల వద్దక్వ కరిగిన బంధ ఉత్తరం.

---

Wave లో పాయింటర్ అనేది `ptr<T>` రూపంలో స్పష్టమైన రకం.
చిరునామా పొందేందుకు `&`, తిరిగి చూడటానికి `deref` ను ఉపయోగించండి.

## ఊహా మరియు ప్రారంభం

```wave
var x: i32 = 10;
var p: ptr<i32> = &x;
```

పాయింటర్ రకం పడుచు ఉండవచ్చు.

```wave
var p1: ptr<i32> = &x;
var p2: ptr<ptr<i32>> = &p1;
```

## దాన్ తిరిగి చూడటం

```wave
var x: i32 = 10;
var p: ptr<i32> = &x;

println("{}", deref p); // 10
deref p = 20;
println("{}", x);       // 20
```

## `null` సాహయ నియమాలు

`null` అనేది **సార్వత్రిక లిటర్లు**. ఇది గుర్తింపు కాదు మరియు చేర్చిన పేరుగా ఉపయోగించలేము.

ముఖ్య నియమాలు:

- `null`ను కేవలం `ptr<T>` లక్ష్యాలలో మాత్రమే కలుపుకోవచ్చు.
- `i32`, `bool`, `array<...>` వంటి శూన్య ఆకరణం కాదు.
- మీకు సులువు వాక్యం ఉండట్లేదు స్పష్టంగా `null` ఉపయోగించండి.

```wave
var p: ptr<i32> = null;
var arrp: ptr<array<i32, 3>> = null;

// var n: i32 = null;  // తప్పు
// var b: bool = null; // తప్పు
```

## పాయింటర్ అంకెల

Wave క్రింది పాయింటర్ అంకెలను మద్దతిస్తుంది.

- `ptr + int`: GEP ఆధారిత పాయింటర్ ముందుకు.
- `int + ptr`: సమాన చర్య.
- `ptr - int`: GEP ఆధారిత పాయింటర్ వెనక్కి.
- `ptr - ptr`: `i64` బైట్ తేడాను లెక్కించండి.

పాయింట్:

- `ptr<T> +/- n` `T` యొక్క పరిమాణం(`sizeof(T)`)ని ప్రాతిపదించుకుని ముందుకు వెళ్ళే దారులు.
- అందువల్ల, `ptr<i32> + 3` బైట్ ప్రకారం `+12` కి వెళ్ళడానికి.

```wave
var base: ptr<i32> = 0x1000 as ptr<i32>;

var p1: ptr<i32> = base + 3; // 0x1000 + 12
var p2: ptr<i32> = 2 + base; // 0x1000 + 8
var p3: ptr<i32> = base - 1; // 0x1000 - 4

var diff: i64 = p1 - base;   // 12 (బైట్ తేడా)
```

## పాయింట్ పోల్చడం

పాయింటర్లు పోలికలో ఉపయోగించవచ్చు.

```wave
if (p == null) { ... }
if (p != null) { ... }
if (p1 == p2) { ... }
```

## అనుబంధం మరియు సంబంధం

పాయింటర్ అక్కులు:

```wave
var a: i32 = 10;
var b: i32 = 20;
var arr: array<ptr<i32>, 2> = [&a, &b];
println("{} {}", deref arr[0], deref arr[1]);
```

అక్కు పాయింటర్:

```wave
var p: ptr<array<i32, 3>> = &[1, 2, 3];
if (p != null) {
    println("{}", deref p[1]);
}
```

## భద్రతా గమనిక

Wave ప్రస్తుతం Rust తరహా అధికారం/అవధి ఆధారిత పాయింటర్ భద్రతా నమూనాగా లేదు.
అందుకే `null` తిరిగి చూడటాన్ని ఆటోమేటిక్ గా అడ్డుకోదు. `deref` ముందు స్పష్టమైన `null` తనిఖీని ఉంచే పద్ధతిని పరిశీలించండి.
