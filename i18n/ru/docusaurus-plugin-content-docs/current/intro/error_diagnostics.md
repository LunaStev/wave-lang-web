# Диагностика ошибок

Компилятор Wave выводит ошибки вместе с кодом ошибки (`E####`), указанием источника/местоположения/решением подсказки все за один раз.

## Формат вывода

Основной формат выглядит следующим образом.

```text
ошибка[E3001]: семантическая проверка не прошла: использование необъявленного идентификатора 'x'
  --> файл.wave:2:18
 1 | fun main() {
 2 |     println("{}", x);
   |                  ^ не найдено в этой области
   = контекст: семантическая проверка
   = помощь: исправьте вопросы изменяемости, области видимости и правил синтаксиса
```

Элементы вывода:

- `ошибка[E....]`: код ошибки и сводка
- `--> файл:строка:столбец`: расположение проблемы
- Блок кода + выделение кареткой(`^`)б
- `контекст`, `ожидалось`, `найдено`, `замечание`, `помощь`, `предложение`

## Коды ошибок директорий

- `E1001` неожиданный символ
- `E1002` незакрытый блочный комментарий
- `E1003` Незамкнутая строка
- `E1004` неправильный escape в строке
- `E1005` Неправильный символьный литерал
- `E1006` Неправильный формат числового литерала
- `E2001` Ошибка синтаксического анализаб
- `E3001` Ошибка семантической проверки
- `E3102` Присвоение `null` непредназначенного для этого указателя
- `E3201` Запрет отрицательного целочисленного уменьшения
- `E9001` Внутренняя ошибка генерации кода в бэкенде

## Ошибки бэкенда также указывают на местоположение исходного кода

Даже если паника компилятора происходит на уровне генерации кода (LLVM), возможное место вызова/объявления указывается в случае возможности.

```text
ошибка[E9001]: внутренняя ошибка компилятора (llvm-ir-generation)
  --> test.wave:12:9
   = найдено: Функция 'foo' не найдена
   = замечение: позиция источника предположена из-за нерешенного имени функции в панике бэкэнда
```

В случае невозможности предположить местоположение используется резервное, это указано в разделе `note`.
